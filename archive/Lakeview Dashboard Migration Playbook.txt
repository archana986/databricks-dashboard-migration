Purpose
Automate migration of an AI/BI (Lakeview) dashboard when catalog and schema names change: retrieve the dashboard definition, update dataset SQL references, update warehouse binding if needed, publish, and validate.


________________


What you’ll automate
* Retrieve and update a Lakeview dashboard’s serialized JSON and warehouse using Lakeview REST APIs (GET/PATCH/POST).
* Optionally export/import the .lvdash.json file through the UI or Workspace APIs for versioning and side-by-side tests.
* Publish the dashboard and, if required, embed credentials using a service principal for broader viewer access.
* Validate each dataset query on the target catalog/schema with the Statement Execution API.


________________


Improved migration steps (Lakeview only)
1. Baseline and backup
* GET the dashboard to capture display_name, etag, warehouse_id, and serialized_dashboard. Keep this payload as your rollback point.
* Optional: Export the .lvdash.json and check it into source control; the UI supports export/import and replace, and dashboards support Git-based versioning flows.
2. Discover all references
* Parse serialized_dashboard to enumerate dataset SQL strings.
* Generate a list of fully qualified identifiers observed in queries (catalog.schema.table) to guide mapping.
3. Define mapping
* Create an explicit mapping (old_catalog, old_schema) → (new_catalog, new_schema) so rewrites are deterministic and traceable.
4. Dry-run rewrite with diffs
* Rewrite only SQL text within serialized_dashboard. Produce a diff/report:
   * Replaced references.
   * Any lingering old_catalog.old_schema to catch misses early.
5. Pre-commit validation
* For each rewritten dataset SQL, call the Statement Execution API against your target warehouse, optionally setting catalog/schema parameters for default scoping. Record success/failure and basic row counts.
6. Choose deployment mode
* In-place (recommended to preserve path and permissions):
   * PATCH /api/2.0/lakeview/dashboards/{dashboard_id} with updated serialized_dashboard, etag, and (optionally) a new warehouse_id.
* Clone (recommended for side-by-side testing or phased cutover):
   * POST /api/2.0/lakeview/dashboards with display_name, parent_path, warehouse_id, and the rewritten serialized_dashboard to create a new object.
   * Alternatively, use UI/Workspace file import/replace for .lvdash.json flows.
7. Publish and share
* Publish the dashboard to make it viewable. If viewers lack direct data access, publish with embed_credentials=true using a service principal token.
8. Validate end-to-end
* Open the published dashboard; verify filters and visuals, expected row counts, and KPIs render correctly.
* Recreate schedules/subscriptions (email/Slack) if needed.
9. Permissions and assets
* Reconfirm share permissions (workspace or account-level).
* If the dashboard embeds images from Volumes, ensure viewers have UC permissions to those files.
10. Audit and rollback
* Use audit logs to confirm create/update/publish events occurred during your change window.
* Roll back by restoring the saved serialized_dashboard or re-importing the backup .lvdash.json.

Robust rewrite and validation blueprint (Python + REST)
Key points:


* Operate only on Lakeview: GET → rewrite → PATCH/POST with ETag for safe updates.
* Validate with the Statement Execution API on your target warehouse.


import os, re, json, time, requests


HOST = os.environ["DATABRICKS_HOST"]      # https://<workspace-url>
TOKEN = os.environ["DATABRICKS_TOKEN"]
WH_ID = os.environ["DBSQL_WAREHOUSE_ID"]  # target SQL warehouse ID
HEADERS = {"Authorization": f"Bearer {TOKEN}", "Content-Type": "application/json"}


# Declarative mapping: (old_catalog, old_schema) -> (new_catalog, new_schema)
CAT_SCHEMA_MAP = {
    ("oldcat", "sales"): ("newcat", "sales_analytics"),
    # add more mappings as needed...
}


DASHBOARD_ID = "04aab30f99ea444490c10c85852f216c"  # your Lakeview dashboard ID
IN_PLACE = True  # True: PATCH in place; False: create a new dashboard


def get_dashboard(did):
    r = requests.get(f"{HOST}/api/2.0/lakeview/dashboards/{did}", headers=HEADERS)
    r.raise_for_status()
    return r.json()  # includes display_name, etag, serialized_dashboard, warehouse_id


def rewrite_sql(sql):
    # Replace backtick-quoted and bare cat.schema.table prefixes
    for (oc, os_), (nc, ns) in CAT_SCHEMA_MAP.items():
        patterns = [
            (rf"`{re.escape(oc)}`\.`{re.escape(os_)}`\.", f"`{nc}`.`{ns}`."),
            (rf"\b{re.escape(oc)}\.{re.escape(os_)}\.", f"{nc}.{ns}."),
        ]
        for pat, rep in patterns:
            sql = re.sub(pat, rep, sql, flags=re.IGNORECASE)
    return sql


def rewrite_serialized(lvdash):
    sd = json.loads(lvdash)
    def walk(x):
        if isinstance(x, dict):
            for k, v in x.items():
                if isinstance(v, str) and k in ("sql", "query", "statement"):
                    x[k] = rewrite_sql(v)
                else:
                    walk(v)
        elif isinstance(x, list):
            for i in range(len(x)):
                walk(x[i])
    walk(sd)
    return json.dumps(sd, separators=(",", ":"), ensure_ascii=False)


def patch_dashboard(did, etag, serialized, name=None):
    body = {"etag": etag, "warehouse_id": WH_ID, "serialized_dashboard": serialized}
    if name: body["display_name"] = name
    r = requests.patch(f"{HOST}/api/2.0/lakeview/dashboards/{did}", headers=HEADERS, json=body)
    r.raise_for_status()
    return r.json()


def create_dashboard(name, parent_path, serialized):
    body = {"display_name": name, "parent_path": parent_path, "warehouse_id": WH_ID, "serialized_dashboard": serialized}
    r = requests.post(f"{HOST}/api/2.0/lakeview/dashboards", headers=HEADERS, json=body)
    r.raise_for_status()
    return r.json()["dashboard_id"]


def publish(did, embed=False):
    r = requests.post(f"{HOST}/api/2.0/lakeview/dashboards/{did}/published", headers=HEADERS, json={"embed_credentials": embed})
    r.raise_for_status()


def run_statement(sql, catalog=None, schema=None):
    # Basic validation call with Statement Execution API
    body = {"warehouse_id": WH_ID, "statement": sql}
    if catalog: body["catalog"] = catalog
    if schema: body["schema"] = schema
    r = requests.post(f"{HOST}/api/2.0/sql/statements/", headers=HEADERS, json=body)
    r.raise_for_status()
    sid = r.json()["statement_id"]
    while True:
        g = requests.get(f"{HOST}/api/2.0/sql/statements/{sid}", headers=HEADERS)
        g.raise_for_status()
        state = g.json()["status"]["state"]
        if state in ("SUCCEEDED", "FAILED", "CANCELED"):
            return state
        time.sleep(1)


def main():
    d = get_dashboard(DASHBOARD_ID)
    rewritten = rewrite_serialized(d["serialized_dashboard"])


    # Optional: Extract dataset SQLs and validate with run_statement(sql, catalog="newcat", schema="sales_analytics")


    if IN_PLACE:
        patch_dashboard(DASHBOARD_ID, d["etag"], rewritten, name=d["display_name"])
        new_id = DASHBOARD_ID
    else:
        parent_dir = d["path"].rsplit("/", 1)[0] if "path" in d else "/Workspace/Users"
        new_id = create_dashboard(d["display_name"] + " (new catalog)", parent_dir, rewritten)


    publish(new_id, embed=False)  # set True and authenticate as service principal if embedding credentials
    print(f"Migration complete. Dashboard: {new_id}")


if __name__ == "__main__":
    main()

Extra tips
* For multiple dashboards, orchestrate this flow in a Lakeflow Job or external CI/CD runner and add a smoke-test step per dashboard using the Statement Execution API.
* Revalidate access to any embedded images stored in Volumes; image file permissions are separate from dashboard sharing.
* Track changes through audit logs to confirm create/update/publish events and facilitate rollbacks.
